\section{Approche 1 : CNN sur Images Brutes}
%%============================================================

\begin{approchebox}[BrickRed]{Approche 1 | Christ Bohou | ECHEC -- val\_accuracy 34.3\%}
Entrainement d'un CNN directement sur des images 64$\times$64 pixels

\subsection{Architecture du modele}

\begin{lstlisting}[caption={Architecture CNN 3 blocs avec BatchNorm et Dropout.}]
modele = models.Sequential([
    # Bloc 1 - contours et textures simples
    layers.Conv2D(32,(3,3),activation='relu',padding='same',
                  input_shape=(64,64,3)),
    layers.BatchNormalization(),
    layers.Conv2D(32,(3,3),activation='relu',padding='same'),
    layers.MaxPooling2D(2,2),
    layers.Dropout(0.25),
    # Bloc 2 - formes complexes
    layers.Conv2D(64,(3,3),activation='relu',padding='same'),
    layers.BatchNormalization(),
    layers.Conv2D(64,(3,3),activation='relu',padding='same'),
    layers.MaxPooling2D(2,2),
    layers.Dropout(0.25),
    # Bloc 3 - configurations de main
    layers.Conv2D(128,(3,3),activation='relu',padding='same'),
    layers.BatchNormalization(),
    layers.MaxPooling2D(2,2),
    layers.Dropout(0.25),
    # Classification
    layers.Flatten(),
    layers.Dense(256, activation='relu'),
    layers.BatchNormalization(),
    layers.Dropout(0.5),
    layers.Dense(5, activation='softmax'),   # 5 classes
])
modele.compile(optimizer='adam',
               loss='sparse_categorical_crossentropy',
               metrics=['accuracy'])
\end{lstlisting}

\textbf{Parametres entraenables : 2\,240\,037.}

Data augmentation : flip horizontal, rotation $\pm$15\%, zoom $\pm$15\%,
luminosite $\pm$20\%, contraste $\pm$20\% -- appliques en ligne sur chaque batch.
EarlyStopping patience=8, ReduceLROnPlateau factor=0.5, split 80/20, batch 32, max 30 epochs.

\subsection{Limite critique}

\begin{failbox}[Distribution shift -- inutilisable en conditions reelles]
Les deux modeles atteignent $\sim$100\% sur Kaggle (fond blanc)
mais \textbf{echouent sur une webcam reelle}.\\
Test concret : le CNN Kaggle predit \texttt{up} a 99.8\% sur une image
aleatoire de bruit pur -- il a appris le fond blanc, pas les mains.
\end{failbox}

Ce phenomene est le \textbf{distribution shift} : l'ecart entre la
distribution d'entrainement (fond blanc controlee) et la distribution
reelle (fond de piece, eclairage variable) empeche toute generalisation.

\begin{figure}[h!]
\centering
\includegraphics[width=\textwidth]{cnn_training/modeles/images_rapport/distribution_shift.png}
\caption{\textbf{Illustration du distribution shift.}
A gauche : le dataset Kaggle -- fond blanc parfaitement uniforme,
eclairage controlee, cadrage identique pour toutes les images.
A droite : les conditions reelles de deploiement -- fond encombre,
ombres, eclairage variable, teinte de peau differente.
Le modele entraine exclusivement sur fond blanc apprend les statistiques
du fond, pas celles de la main. En conditions reelles, il ne voit
jamais de fond blanc et echoue systematiquement.}
\end{figure}

\subsection{Collecte des donnees}

\subsubsection{Normalisation geometrique}

Pour chaque frame, le vecteur brut est normalise :

\begin{equation}
\tilde{\mathbf{p}}_i =
\frac{\mathbf{p}_i - \mathbf{p}_0}
{\max_{j} \|\mathbf{p}_j - \mathbf{p}_0\|_\infty}
\end{equation}

$\mathbf{p}_0$ = poignet. Resultat : $\tilde{\mathbf{p}} \in [-1,1]^{63}$,
sans information de fond, couleur ou taille.

\begin{lstlisting}[caption={Normalisation des landmarks.}]
def normaliser(landmarks):
    pts = np.array([[lm.x, lm.y, lm.z] for lm in landmarks],
                   dtype=np.float32)
    pts -= pts[0]           # centrage sur le poignet
    s = np.max(np.abs(pts))
    if s > 0: pts /= s     # mise a l'echelle [-1, 1]
    return pts.flatten()   # vecteur de 63 valeurs
\end{lstlisting}

\textbf{Cette normalisation est rigoureusement identique
lors de la collecte, de l'entrainement et de l'inference.}

Volume : environ 2\,100 echantillons -- 7 minutes de capture par classe.

\subsection{Integration dans le jeu}

\begin{lstlisting}[caption={Boucle d'inference temps reel.}]
SEUIL_CONFIANCE = 0.70

while True:
    ret, frame = cap.read()
    frame = cv2.flip(frame, 1)
    rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
    result = landmarker.detect_for_video(
        mp.Image(image_format=mp.ImageFormat.SRGB, data=rgb),
        timestamp_ms)
    if result.hand_landmarks:
        vecteur = normaliser(result.hand_landmarks[0]).reshape(1,-1)
        probs = model.predict(vecteur, verbose=0)[0]
        idx = np.argmax(probs)
        if probs[idx] >= SEUIL_CONFIANCE:
            geste = CLASSES[idx]
            if geste != 'NEUTRAL':
                commande_queue.put_nowait({'type': geste})
\end{lstlisting}

Architecture bi-thread : le thread principal gere la video et l'inference,
un thread dedie gere l'envoi WebSocket sans bloquer la capture.
Un hold timer de 180\,ms dans Node.js maintient la touche enfoncee
entre deux commandes consecutives.

\paragraph{FIRE moins robuste avec la main droite.}
Donnees FIRE collectees principalement avec la main gauche.

que le modele ne reconnait pas toujours.

\paragraph{Impossibilite de tirer en se deplacant.}
Une seule commande par frame -- LEFT et FIRE simultanement impossible.

\section{Perspectives}
%%============================================================

\textbf{Classes composites.}
Ajouter \texttt{LEFT\_FIRE} et \texttt{RIGHT\_FIRE} (poing incline)
pour envoyer deux commandes simultanement.

\textbf{FIRE deux mains.}
Recollecte avec les deux mains ou augmentation par symetrie ($x_i \to -x_i$).

\textbf{LSTM temporel.}
Sequences de 5 a 10 frames pour gestes dynamiques
et elimination des ambiguites statiques.

%%============================================================
\section{Conclusion}
%%============================================================

\begin{enumerate}
\item \textbf{La representation est decisive.}
      Pixels bruts $\to$ landmarks : dimension $\div 195$,
      parametres $\div 66$, accuracy $34\% \to 99.6\%$.

\item \textbf{Le score de validation ne garantit pas la generalisabilite.}
      100\% sur Kaggle, 0\% en conditions reelles.

\item \textbf{L'ingenierie systeme compte autant que le modele.}
      Threading, hold timer, cooldowns sont indispensables
      pour une experience de jeu fluide.
\end{enumerate}
